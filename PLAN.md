# PLAN: AI Secretary (FastAPI + OpenAI + PostgreSQL)

## Принципы выполнения

- Следовать `SKILLS.md`: чистая модульная архитектура, разделение слоев (`API`, `Services`, `Tools`, `Database`).
- Не размещать бизнес-логику в роуте `POST /agent`.
- Для всех публичных функций: type hints, логирование, обработка ошибок.
- Инструменты должны быть отдельными вызываемыми функциями с явным входом/выходом.

## Этап 1. Базовый каркас проекта

1. Создать структуру:
   - `app/main.py`
   - `app/db.py`
   - `app/models.py`
   - `app/tools/tasks.py`
   - `app/tools/memory.py`
   - `app/services/agent.py`
2. Добавить `__init__.py` в `app/`, `app/tools/`, `app/services/`.
3. Подготовить минимальный `README.md` с инструкцией запуска API.

## Этап 2. Конфигурация и подключение БД

1. Реализовать в `app/db.py`:
   - чтение `DATABASE_URL` из переменных окружения;
   - создание SQLAlchemy engine/session;
   - базовый dependency для FastAPI (если потребуется расширение API).
2. Подготовить основу для миграций (структура моделей без хардкода логики в одном файле).
3. Добавить обработку ошибок подключения к БД и логирование.

## Этап 3. Модели данных PostgreSQL

1. Описать в `app/models.py` таблицы:
   - `users`
   - `tasks`
   - `memory`
2. Для `tasks` добавить ключевые поля:
   - `id`, `user_id`, `title`, `description`, `status`, `due_at`, `created_at`.
3. Для `memory` добавить ключевые поля:
   - `id`, `user_id`, `content`, `kind`, `created_at`.
4. Добавить связи и индексы, где это полезно для поиска задач/памяти.

## Этап 4. Реализация Tools слоя

1. В `app/tools/tasks.py` реализовать:
   - `create_task`
   - `get_tasks`
2. В `app/tools/memory.py` реализовать:
   - `save_memory`
   - `search_memory`
3. Для каждой tool-функции:
   - понятные входные параметры;
   - предсказуемый JSON-совместимый результат;
   - логирование вызова и результата;
   - аккуратная обработка исключений.

## Этап 5. Agent service (OpenAI function calling)

1. В `app/services/agent.py` реализовать сервис-оркестратор:
   - принимает `user_id` и `message`;
   - отправляет запрос в OpenAI;
   - регистрирует доступные tools (`create_task`, `get_tasks`, `save_memory`, `search_memory`);
   - выполняет выбранный tool и возвращает финальный текстовый ответ.
2. Добавить защиту от невалидных аргументов tools.
3. Добавить централизованное логирование шагов агента.

## Этап 6. API слой

1. В `app/main.py` поднять FastAPI приложение.
2. Описать Pydantic-схемы:
   - Request: `{ "user_id": "string", "message": "string" }`
   - Response: `{ "response": "string" }`
3. Реализовать `POST /agent`:
   - валидация входа;
   - вызов `AgentService`;
   - возврат результата в требуемом формате.
4. Добавить health-check endpoint (например, `GET /health`) для удобства интеграции с n8n.

## Этап 7. Надежность и наблюдаемость

1. Настроить единый logger для приложения.
2. Добавить обработчики ошибок:
   - ошибки валидации;
   - ошибки OpenAI;
   - ошибки БД;
   - fallback-ответы.
3. Проверить, что API возвращает понятные и стабильные ответы для n8n.

## Этап 8. Проверка сценариев

1. Проверить базовые сценарии вручную:
   - создание задачи;
   - получение списка задач;
   - сохранение памяти;
   - поиск по памяти;
   - обычный вопрос без вызова tool.
2. Проверить негативные сценарии:
   - пустой `message`;
   - несуществующий `user_id`;
   - недоступная БД;
   - ошибки OpenAI.

## Этап 9. Интеграционный контракт с n8n/Telegram

1. Зафиксировать стабильный контракт `POST /agent`.
2. Описать ожидаемые таймауты, формат ошибок и ретраи.
3. Подготовить примеры запросов для n8n в формате PowerShell.

## Definition of Done (MVP)

- Работает endpoint `POST /agent` и возвращает осмысленный `response`.
- Реализованы 4 обязательные tools:
  - `create_task`
  - `get_tasks`
  - `save_memory`
  - `search_memory`
- Данные сохраняются в PostgreSQL (`users`, `tasks`, `memory`).
- Агент использует OpenAI function calling для выбора инструмента.
- Логирование и базовая обработка ошибок присутствуют во всех слоях.
